二、springboot 配置

可以通过修改配置文件，来修改springboot默认配置

1、配置文件形式
  • application.properties
  • application.yml
2、将配置文件中的的值映射到组件
（1）@ConfigurationProperties(prefix = "xxx") 注解
  • 将配置文件中的属性绑定的标有该注解的类
  • 默认加载全局配置文件
  • 支持 @validation

application.yml

application.properties 配置文件（二选一）

  • idea配置文件utf-8
  • properties 默认GBK

javaBean

导入配置文件处理器，以后编写配置就有提示了

（2）@Value注解
@ConfigurationProperties @Value
功能 批量注入配置文件属性 单个指定
松散绑定(语法) 支持 不支持
spEL 不支持 支持
JSR303校验 支持 不支持
复杂类型 支持 不支持

松散语法：javaBean中last-name(或者lastName) –>application.properties中的last-name;

spEL语法：#{11*2}

JSR303：@Value会直接忽略，校验规则

JSR303校验：

（3）@PropertySource(value= “classpath:person.properties”)
  • 使用自己写的properties文件

（4）@ImportResource(locations=“classpath: xxx.xml”)

  • 导入spring配置文件，使spring配置文件中的内容生效
  • spring 推荐用配置类配置，全注解方式配置
（5）spring推荐的配置方式
  • 使用配置类，全注解方式。
  • @Configuration 指明配置类
  • @Bean 将标注的方法的返回值添加到容器中。

/**

  • @Configuration：指明当前类是一个配置类；就是来代替之前的Spring配置文件
    *
  • 在配置文件中用标签添加组件
    */

4、springboot test
5、配置文件占位符
  • 可以在配置文件中使用前面配置过的属性
  • $app.name: 默认值 来指定没有配置时的默认值
（1）随机数
  • 冒号指定取不到值时的默认值
6、Profile
（1）多profile 文件
  • application-{profile}.properties/yml

配置文件名可以是 application-dev.properties

然后在application.properties中激活

(2) yml支持多文档快模式
  • —分割文档块
（6）命令行方式激活profiles

  • 运行jar包时输入

用虚拟机参数激活

7、配置文件的优先级
  • 与路径有关，高优先级覆盖低优先级，由高到低排列依次为
    • file: ./config
    • file: ./
    • classpath: /config/
    • classpath: /
  • 可以通过修改spring.config.location来改变默认配置
